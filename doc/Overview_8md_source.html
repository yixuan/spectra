1 **Spectra** stands for <strong>Sp</strong>arse <strong>E</strong>igenvalue
2 <strong>C</strong>omputation <strong>T</strong>oolkit
3 as a <strong>R</strong>edesigned <strong>A</strong>RPACK.
4 It is a C++ library for large scale eigenvalue
5 problems, built on top of [Eigen](http://eigen.tuxfamily.org),
6 an open source linear algebra library.
7 
8 **Spectra** is implemented as a header-only C++ library, whose only dependency,
9 **Eigen**, is also header-only. Hence **Spectra** can be easily embedded in
10 C++ projects that require calculating eigenvalues of large matrices.
11 
12 The development page of **Spectra** is at
13 [https://github.com/yixuan/spectra/](https://github.com/yixuan/spectra).
14 
15 ## Relation to ARPACK
16 
17 [ARPACK](http://www.caam.rice.edu/software/ARPACK/) is a software written in
18 FORTRAN for solving large scale eigenvalue problems. The development of
19 **Spectra** is much inspired by ARPACK, and as the whole name indicates,
20 **Spectra** is a redesign of the ARPACK library using C++ language.
21 
22 In fact, **Spectra** is based on the algorithms described in the
23 [ARPACK Users' Guide](http://www.caam.rice.edu/software/ARPACK/UG/ug.html),
24 but it does not use the ARPACK code, and it is **NOT** a clone of ARPACK for C++.
25 In short, **Spectra** implements the major algorithms in ARPACK,
26 but **Spectra** provides a completely different interface, and it does not
27 depend on ARPACK.

29 ## Common Usage
30 
31 **Spectra** is designed to calculate a specified number (\f$k\f$) of eigenvalues
32 of a large square matrix (\f$A\f$). Usually \f$k\f$ is much less than the size of matrix
33 (\f$n\f$), so that only a few eigenvalues and eigenvectors are computed, which
34 in general is more efficient than calculating the whole spectral decomposition.
35 Users can choose eigenvalue selection rules to pick up the eigenvalues of interest,
36 such as the largest \f$k\f$ eigenvalues, or eigenvalues with largest real parts,
37 etc.
38 
39 To use the eigen solvers in this library, the user does not need to directly
40 provide the whole matrix, but instead, the algorithm only requires certain operations
41 defined on \f$A\f$, and in the basic setting, it is simply the matrix-vector
42 multiplication. Therefore, if the matrix-vector product \f$Ax\f$ can be computed
43 efficiently, which is the case when \f$A\f$ is sparse, **Spectra**
44 will be very powerful for large scale eigenvalue problems.
45 
46 There are two major steps to use the **Spectra** library:
47 
48 1. Define a class that implements a certain matrix operation, for example the
49 matrix-vector multiplication \f$y=Ax\f$ or the shift-solve operation
50 \f$y=(A-\sigma I)^{-1}x\f$. **Spectra** has defined a number of
51 helper classes to quickly create such operations from a matrix object.
52 See the documentation of Spectra::DenseGenMatProd, Spectra::DenseSymShiftSolve, etc.
53 2. Create an object of one of the eigen solver classes, for example
54 Spectra::SymEigsSolver for symmetric matrices, and Spectra::GenEigsSolver
55 for general matrices. Member functions of this object can then be called to
56 conduct the computation and retrieve the eigenvalues and/or eigenvectors.
57 
58 Below is a list of the available eigen solvers in **Spectra**:
59 - \link Spectra::SymEigsSolver SymEigsSolver \endlink: for real symmetric matrices
60 - \link Spectra::GenEigsSolver GenEigsSolver \endlink: for general real matrices
61 - \link Spectra::SymEigsShiftSolver SymEigsShiftSolver \endlink: for real symmetric matrices using the shift-and-invert mode
62 - \link Spectra::GenEigsRealShiftSolver GenEigsRealShiftSolver \endlink: for general real matrices using the shift-and-invert mode,
63 with a real-valued shift
64 - \link Spectra::GenEigsComplexShiftSolver GenEigsComplexShiftSolver \endlink: for general real matrices using the shift-and-invert mode,
65 with a complex-valued shift

67 ## Examples
68 
69 Below is an example that demonstrates the use of the eigen solver for symmetric
70 matrices.
71 
72 ~~~~~~~~~~{.cpp}
73 #include <Eigen/Core>
74 #include <SymEigsSolver.h> // Also includes <MatOp/DenseSymMatProd.h>
75 #include <iostream>
76 
77 using namespace Spectra;
78 
79 int main()
80 {
81  // We are going to calculate the eigenvalues of M
82  Eigen::MatrixXd A = Eigen::MatrixXd::Random(10, 10);
83  Eigen::MatrixXd M = A + A.transpose();
84 
85  // Construct matrix operation object using the wrapper class DenseGenMatProd
86  DenseSymMatProd<double> op(M);
87 
88  // Construct eigen solver object, requesting the largest three eigenvalues
89  SymEigsSolver< double, LARGEST_ALGE, DenseSymMatProd<double> > eigs(&op, 3, 6);
90 
91  // Initialize and compute
92  eigs.init();
93  int nconv = eigs.compute();
94 
95  // Retrieve results
96  Eigen::VectorXd evalues;
97  if(eigs.info() == SUCCESSFUL)
98  evalues = eigs.eigenvalues();
99 
100  std::cout << "Eigenvalues found:\n" << evalues << std::endl;
101 
102  return 0;
103 }
104 ~~~~~~~~~~
105 
106 And here is an example for user-supplied matrix operation class.
107 
108 ~~~~~~~~~~{.cpp}
109 #include <Eigen/Core>
110 #include <SymEigsSolver.h>
111 #include <iostream>
112 
113 using namespace Spectra;
114 
115 // M = diag(1, 2, ..., 10)
116 class MyDiagonalTen
117 {
118 public:
119  int rows() { return 10; }
120  int cols() { return 10; }
121  // y_out = M * x_in
122  void perform_op(double *x_in, double *y_out)
123  {
124  for(int i = 0; i < rows(); i++)
125  {
126  y_out[i] = x_in[i] * (i + 1);
127  }
128  }
129 };
130 
131 int main()
132 {
133  MyDiagonalTen op;
134  SymEigsSolver<double, LARGEST_ALGE, MyDiagonalTen> eigs(&op, 3, 6);
135  eigs.init();
136  eigs.compute();
137  if(eigs.info() == SUCCESSFUL)
138  {
139  Eigen::VectorXd evalues = eigs.eigenvalues();
140  std::cout << "Eigenvalues found:\n" << evalues << std::endl;
141  }
142 
143  return 0;
144 }

147 ## Shift-and-invert Mode
148 
149 When it is needed to find eigenvalues that are closest to a number \f$\sigma\f$,
150 for example to find the smallest eigenvalues of a positive definite matrix
151 (in which case \f$\sigma=0\f$), it is advised to use the shift-and-invert mode
152 of eigen solvers.
153 
154 In the shift-and-invert mode, selection rules are applied to \f$1/(\lambda-\sigma)\f$
155 rather than \f$\lambda\f$, where \f$\lambda\f$ are eigenvalues of \f$A\f$.
156 To use this mode, users need to define the shift-solve matrix operation. See
157 the documentation of Spectra::SymEigsShiftSolver for details.
